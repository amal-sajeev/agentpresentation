15 / COST REQUIREMENT §1.4 · PARTIAL

Cheap by construction, not by luck

Cost control is built into the architecture — the right model tier for each job, capped loops, conditional pipelines, and lean context. Four levers already in market_research.

Right model for the job

The bulk of agents run on the cheap tier; only evaluators pay for the stronger model. worker_model = gemini-3.1-flash-lite vs critic_model = gemini-3.1-flash.

~90% of calls on the lowest-cost tier

Bounded critic loops

Every quality loop is hard-capped and exits deterministically — a stuck agent can never burn tokens forever. LoopAgent(max_iterations=5) + EscalationChecker.

no runaway re-generation

Conditional pipelines

Whole sub-pipelines are skipped when their input is absent — the sales branch never runs without a named target. needs_sales_intelligence gate → skip_sales.

~73% fewer tokens on no-target runs

Lean, scoped context

Prompt-builders hand each agent only the state fields it needs as compact JSON — not the whole transcript — keeping every context window small.

minimal prompt size per call
